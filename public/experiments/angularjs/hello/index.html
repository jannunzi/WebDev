Example Explained

Declare an AngularJS Application

AngularJS extends the HTML language by declaring several new attributes called directives. AngularJS applications are declared in HTML by using the ng-app directive. This directive declares the HTML document as part of an AngularJS application. The name declared by the directive must match the name of an AngularJS module implemented in a separate JavaScript file.

Below is an example use of the ng-app directive declaring an application called HelloWorldApp. A separate JavaScript file will implement the application by declaring a module of the same name. 

<html lang="en" ng-app="HelloWorldApp">
...
</html>

Load the AngularJS Library

Use a script element to load the AngularJS library. Here's an example of loading the library from a public CDN. 

<html lang="en" ng-app="HelloWorldApp">
    ...
    <script src="https://cdnjs.cloudflare.com/ajax/libs/angular.js/1.5.0-beta.0/angular.js"></script>
    ...
</html>

Implement the AngularJS Application

Implement the application is a separate JavaScript file and reference it from the HTML document using the script element. Below is an example script element loading an AngularJS application implemented in a source file called textToPhrase.js. 

<html lang="en" ng-app="HelloWorldApp">
    ...
    <script src="https://cdnjs.cloudflare.com/ajax/libs/angular.js/1.5.0-beta.0/angular.js"></script>
    <script src="textToPhrase.js"></script>
    ...
</html>

The AngularJS library declares a global object called angular that allows creating applications, configurations, controllers, and other AngularJS artifacts that constitute an AngularJS application.

An AngularJS application is implemented using the .module() method. The .module() method takes two arguments. The first argument is a string denoting the name of the application. The second argument is an array of other modules the application depends on. The array can be empty if the application does not depend on other modules. There can be any number of modules depending on other modules in a dependency hierarchy. The module declared in ng-app is the one module that represents the application.

Below is an example use of the .module() function that declares a module called "HelloWorldApp". The empty array denotes that the module does not depend on any other module. Since the name of the module matches the ng-app="HelloWorldApp" declared in the HTML, this module implements the application referred to in the HTML. The code is in textToPhrase.js 

var app = angular.module("HelloWorldApp", []);

Declare a Controller for an HTML Area

The HTML in an AngularJS application is considered the view of a Web application. AngularJS controllers are responsible for providing data and logic that the a view renders dynamically. Controllers can be responsible for an entire page or a subset of a page. They can be bound to a specific HTML element which will the scope of the controller. HTML outside the element bound to a controller is outside of the scope of the controller.

Below is an example controller named HelloWorldController bound to a DIV element. The controller can provide data that the view can render dynamically. The controller is implemented as a function declared in a separate JavaScript file. 

<html lang="en" ng-app="HelloWorldApp">
    ...
    <script src="https://cdnjs.cloudflare.com/ajax/libs/angular.js/1.5.0-beta.0/angular.js"></script>
    <script src="textToPhrase.js"></script>
    ...
    <div class="container" ng-controller="HelloWorldController">
        ...
    </div>
    ...
</html>

Implement the Controller

Use the .controller() method to name and declare a function that implements the controller. The .controller() method takes two arguments. The first argument is a string used to name the controller. The second argument is the function that implements the controller. The name of the controller must match the name used in the HTML that references it. In the example below the name of the controller is HelloWorldController which matches the name in the HTML ng-controller="HelloWorldController". 

var app = angular.module("HelloWorldApp", []);

app.controller("HelloWorldController", TheController);

function TheController() {
    ...
}

Controller Sends Data to View

The controller is responsible for controlling the view by interacting with user events, implementing client side logic, retrieving data from the client, client side navigation, and retrieving data from the server, and providing data to the client.

To provide data to the client browser, the controller can request an object representation of the HTML it is responsible for, a representation of its scope of influence. To do this, the function that implements the controller declares a variable specifically called $scope. The function can then refer to named placeholders and objects declared in view. For instance, the code below refers to a variable called hello that is declared in the view. 

var app = angular.module("HelloWorldApp", []);

app.controller("HelloWorldController", TheController);

function TheController() {
    ...
    $scope.hello = "Hello World from AngularJS";
    ...
}

The view in turn can declare the variable with the same name as the controller as a placholder where the value of the variable will render as plain text 

<html lang="en" ng-app="HelloWorldApp">
    ...
    <script src="https://cdnjs.cloudflare.com/ajax/libs/angular.js/1.5.0-beta.0/angular.js"></script>
    <script src="textToPhrase.js"></script>
    ...
    <div class="container" ng-controller="HelloWorldController">
        ...
        {{hello}}
        ...
    </div>
    ...
</html>
