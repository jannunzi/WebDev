Setting Up Amazon Web Services

OVERVIEW: Amazon Web Services (AWS) is a platform as a service that allows creating hosted instances of HTTP servers and databases. We will use AWS to host our Web applications online. This course is hosted at OpenShift as a Node.js Web application. This lecture covers how to create simple AWS Node.js Web applications.

Create an Account at Amazon Web Services

Search for 'Amazon Web Services' yields



Navigate to Amazon Web Services and create a free account: https://aws.amazon.com/free.



Create a new account or use your existing Amazon login



Provide billing information



Verify your identity via phone



Choose the basic support plan



Complete the registration process and launch the management console

Create an EC2 Instance

After clicking on Launch Management Console you'll see the list of all the services provided by Amazon Web Services



You'll need to create a group first

After creating a group, you'll be redirected to the management console

From the console, create an EC2 instance (Elastic Compute Cloud). Click Services and then EC2.

In the EC2 Dashboard, click on Launch Instance to create a new EC2 instance.

Select the Amazon Machine Image.

Select a Free instance instance type

Launch your new instance

Create a security key

Download the security key

Launch the instance

View the new instance in the dashboard

Connect to the new instance

If you are on windows, you can use Putty to connect to the remote machine

If you are on windows, download and install Putty to connect to the remote machine

If you are on a Mac, or other Unix machines, use the SSH command from the terminal

You can install Node.js on your new instance, or create a brand new instance with Node.js installed on it. Search for Node.js on Amazon's AWS Marketplace.

Choose on what machine instance you want to install Node.js and accept the terms of service

Wait for the software to be installed on your instance

After a while you'll be notified that your software has been installed

Use SSH to connect to the remote machine and use the node command line to test whether node has been installed properly

You will need to configure the machine's security to open up ports such as port 3000
