CSS - Cascading Style Sheets

OVERVIEW: CSS, or Cascading Stylesheets, is a simple language used to format the look and feel of a Web page. Whereas HTML formats the overall structure and logical organization of a Web page, CSS formats how the page looks. CSS formats the font to use, the colors of text, background, and borders, the white spaces between various elements, and page layout.

CSS Style Attribute and Element



The easiest way to configure the look and feel of content is to use the style attribute. Style attributes allow applying CSS style properties to specific HTML elements. You can configure properties such as foreground and background color, font, borders, padding, and other white space properties. Although easy to use, it is generally considered a bad practice to use the style attribute indiscrimitally since it makes it harder to make style changes across several related pages or Web site wide changes.

CSS Order of Precedence



Content look and feel can be configured in various different ways. Using the style attribute, style element, IDs, classes, document structure, and linked stylesheets. With so many places to declare style rules, care must be taken that they cooperate instead of competing for the final look and feel of the content. Generally, the closer and specific the style rule is for a matching element(s), the more its relevance relative to other rules that might apply to the same element(s). Linking to a CSS Style Sheet



CSS style can be declared outside of the HTML document using the link element. The example below loads two CSS style files located in a css folder. 

<link href="css/style1.css" rel="stylesheet"/>
<link href="css/style2.css" rel="stylesheet"/>

The rel attribute establishes the relationship of the file with respect to the HTML document loading the page. It declares the file as the stylesheet for the document
