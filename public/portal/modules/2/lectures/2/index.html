Box Model

OVERVIEW: The content in a Web page basically consists of various rentangular regions that can render with the content flow, render vertically or horizontally, render with a specific width, height, border, and white space based on various rules and sizes of the content. Collectively these rules are captured as a "box model". The Box Model lecture covers the various properties that govern the box model.

The CSS Box Model



The content in HTML documents (Web pages) consists of a collection of rectangular areas, or boxes, layed out to cover the browser window. The white spaces between these boxes, how they flow and wrap, and how they interact with one another is collectively referred to as the "Box Model".

The box model controls the look-and-feel and behaviour of the content on a page. Three of the more common CSS properties that control the box model are: border, margin, and padding. These three properties control the white space within and between HTML block elements.

This paragraph's border has been configured to have a 10px wide, red dotted border. Applying this styling can be accomplished in various ways. We can use the style attribute right on the paragraph element: style="border: 10px red dotted", but inline styles are generally frowned upon. Instead we can decalre a CSS rule in the style element in the head element. The following CSS rule would do the trick: 

p {
  border: 10px red dotted;
}

Controlling Element Position with CSS



Browsers parse HTML documents they download from the Web and render the content as stipulated by the HTML elements and style it based on the CSS style sheet. Elements render from top to bottom and left to right, without overlapping each other. The CSS position property allows overriding the default position behavior of HTML elements. The position property can have one of the following values:

  • static is the default value for HTML elements and elements are not affected by position properties top, bottom, left, or right
  • relative positions elements relative to their default position. So if an element normally ends up rendering at top : 100px, and left : 200px, then position attributes such as top, bottom, left, or right will nudge the element in that direction
  • fixed positions elements relative to the browser window at particular location and it stays there even if the page scrolls
  • absolute positions elements relative to the nearest parent that is not static

Controlling Element Visibility with CSS



By default elements are rendered at some physical area of the browser window. Their content is visible within a determined rectangular area of the window. CSS allows controlling whether an element is actually rendered or not. There are two basic ways to do this; either with the visibility CSS property or the display property. The visibility property can have one of the following values:

  • visible is the default value and makes the content visible
  • hidden does not render the content, but the rectangular area that the content would have occupied is still visible. In other words the content is invisible, but we know of its presence because of it's surroundings. Like a blackhole, we can't see it, but we know it's there because we see something is missing :)
  • collapse is used hide an entire table row
  • initial resets the visibility to what ever it was by default before it might have changed, perhaps defined by a parent element
  • inherit derives the value of the visibility property from its parent element

Controlling Element Depth with CSS



Browsers parse HTML documents they download from the Web and render the content as stipulated by the HTML elements and style it based on the CSS style sheet. Elements render from top to bottom and left to right, without overlapping each other. Using the CSS position property it is possible to remove HTML elements from the normal rendering order and even force elements to overlap others. When elements overlap, document order is used to decide which elment renders on top of the other. Document order is the order in the HTML file document where the element is declared. That is, HTML elements declared earlier in the HTML document have lower document order than HTML elements declared later in the same HTML document. Lower order elements are rendered earlier than elements with higher element order, therefore, later elements render above earlier elements.

The CSS property z-index allows to override this behavior and declare a specific rendering order. By default all HTML elements have the same z-index of 0 and document order is used to decide order of overlapping elements. Elements with a higher z-index will render above elements with lower z-index. For instance, the two div elements below don't have a z-index so div#one renders below div#two if they overlap. 

<div id="one"></div>
<div id="two"></div>

z-index can override the default rendering order. The example below uses z-index to render div#one renders above div#two if they overlap. 

<style>
  div#one {
    z-index : 2;
  }
  div#two {
    z-index : 1;
  }
</style>
